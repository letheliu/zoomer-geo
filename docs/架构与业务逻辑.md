# Zoomer GEO 技术架构与业务逻辑

## 一、项目概述

Zoomer GEO 是一套 **生成式引擎优化（Generative Engine Optimization）** 系统，帮助品牌在 AI 搜索引擎（如 DeepSeek、豆包、通义千问、文心一言）中获得更高的引用率和可见度。

### 核心目标

- **监测**：追踪品牌在各 AI 平台的被引用情况
- **优化**：将内容改写为"AI 友好"形态，提高被引用概率
- **结构化**：生成 Schema.org 标记和 llms.txt，让 AI 更容易理解和引用
- **图谱**：构建品牌知识图谱，增强实体识别

---

## 二、为什么 GEO 能起作用？

### 2.1 AI 搜索的引用机制

当用户在 AI 搜索引擎中提问时，AI 会：

1. **检索**：从互联网抓取相关内容（RAG 检索增强生成）
2. **理解**：解析内容的结构、实体、关系
3. **生成**：综合多个来源生成答案
4. **引用**：在答案中标注信息来源

### 2.2 GEO 的优化逻辑

传统 SEO 优化的是"网页在搜索引擎结果页的排名"，GEO 优化的是"内容被 AI 引用的概率"。

**AI 偏好引用的内容特征**：

| 特征 | 说明 | 示例 |
|------|------|------|
| 数据锚点 | 包含具体数字、时间、统计 | "2024年用户增长300%" |
| 实体明确 | 主语、谓语、宾语完整 | "Zoomer.top 是一款AI协作工具" |
| 定义句 | 开头有清晰定义 | "GEO是一种优化AI引用的技术" |
| 独立可理解 | 段落可独立被引用，不依赖上下文 | 自包含的段落 |
| 结构化标记 | Schema.org JSON-LD | `SoftwareApplication` 类型 |
| 机器可读协议 | llms.txt | 告诉 AI 如何理解你的站点 |

### 2.3 本系统的优化路径

```
原始内容 → 原子化切分 → AI友好性评分 → LLM改写 → 结构化标记 → 监测效果
    ↓           ↓           ↓           ↓           ↓           ↓
  长文本    语义段落    缺失维度分析   补充锚点    JSON-LD     SOV分数
```

## 四、核心模块详解

### 4.1 引用监测（Citation Monitor）

**原理**：模拟用户向 AI 平台提问，分析回答中是否提及品牌。

**流程**：
1. 从查询库获取监测词（如 "AI设计工具推荐"）
2. 调用各 AI 平台 API（DeepSeek、豆包、千问、文心）
3. 分析返回结果：
   - 品牌是否被提及
   - 在答案中的排名位置
   - SOV（Share of Voice）分数
4. 记录引用事件到数据库

**SOV 计算公式**：
```
SOV = 品牌提及次数 / 所有品牌提及总次数
```

**平台适配器**：
- `DeepSeekAdapter`: 调用 DeepSeek API
- `DoubaoAdapter`: 调用火山引擎豆包 API
- `QwenAdapter`: 调用阿里云通义千问 API
- `ErnieAdapter`: 调用百度文心一言 API

### 4.2 内容优化（Content Optimizer）

**原理**：将长文本切分为"原子化段落"，评估 AI 友好性，自动改写低分段落。

**流程**：
1. **原子化（Atomize）**：用 LLM 将长文本切分为语义段落
2. **评分（Score）**：评估每个段落的 AI 友好性
3. **改写（Rewrite）**：对低分段落进行 LLM 改写
4. **FAQ 生成**：为内容自动生成问答对

**评分维度**（满分100）：
| 维度 | 权重 | 说明 |
|------|------|------|
| 数字锚点 | 35% | 包含具体数字、时间、统计数据 |
| 实体锚点 | 25% | 包含机构名、专有名词 |
| 独立可理解 | 25% | 主语+谓语+宾语完整 |
| 定义句 | 15% | 开头有"X是Y"格式定义 |

**阈值**：70分以下触发改写

### 4.3 Schema 生成（Schema Generator）

**原理**：自动生成结构化数据标记，帮助 AI 理解页面内容。

**输出**：
1. **JSON-LD**：符合 Schema.org 规范的结构化数据
   - `SoftwareApplication`：软件产品
   - `Organization`：组织机构
   - `FAQPage`：常见问题
   - `Article`：文章
2. **llms.txt**：AI 爬虫协议文件
   - 品牌定义
   - 核心产品链接
   - 常见问答
   - 更新频率

### 4.4 知识图谱（Knowledge Graph）

**原理**：从内容中抽取实体和关系，构建品牌知识图谱。

**流程**：
1. **实体抽取**：用 LLM 从文本中提取实体（产品、功能、人物等）
2. **关系抽取**：识别实体间关系（"产品A包含功能B"）
3. **去重入库**：避免重复实体
4. **导出**：支持 JSON/RDF 格式导出

## 五、数据流

### 5.1 监测流程
```
查询库 → 平台适配器 → AI平台API → 结果分析 → 引用事件入库 → Dashboard展示
```

### 5.2 优化流程
```
内容页面 → 原子化 → 评分 → 低分段落改写 → 任务审核 → 发布 → 重新生成Schema
```

### 5.3 闭环逻辑
```
监测发现"某query未被引用" → 分析应优化的内容 → LLM生成改写方案 → 人工审核 → 发布 → 监测效果变化
```

## 六、技术栈

| 层级 | 技术 | 用途 |

| 前端 | Vue 3 + Naive UI + Vite | 管理界面 |

## 七、使用指南

### 7.1 初始化
1. 注册 Workspace，获取 API Key
2. 配置 AI 平台凭证（设置页面）
3. 配置 LLM 服务（.env 文件）

### 7.2 监测流程
1. **添加监测词**：在查询库手动添加或用 LLM 批量生成
2. **触发监测**：在引用报告页面选择平台触发
3. **查看结果**：Dashboard 查看 SOV 分数，引用报告查看详情

### 7.3 优化流程
1. **添加内容**：在内容页面添加要优化的页面
2. **运行优化**：系统自动原子化、评分、生成改写任务
3. **审核发布**：在任务列表审核改写方案，发布后自动更新 Schema

## 八、GEO 效果评估

### 关键指标
- **SOV 分数**：品牌在 AI 答案中的声音份额
- **引用率**：被 AI 引用的查询比例
- **引用排名**：在 AI 答案中的位置
- **内容 AI 友好度**：原子化段落的平均评分

### 优化周期
1. **Baseline**：初始监测，建立基准数据
2. **优化**：改写低分内容，补充结构化标记
3. **验证**：重新监测，对比效果
4. **迭代**：持续优化，形成闭环

---

## 九、注意事项

1. **人工审核必须**：AI 改写的内容 100% 需要人工审核，避免事实错误
2. **持续迭代**：AI 模型版本更新频繁，优化策略需持续调整
3. **避免黑帽**：不要尝试 prompt injection、训练数据污染等手段
4. **数据安全**：API Key 妥善保管，不要提交到代码仓库
